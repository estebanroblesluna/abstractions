  Desarrollo de software dirigido por modelos: teorías, metodologías y herramientas
  Objetivos
  El objetivo general de este proyecto es contribuir al mejoramiento de los procesos de desarrollo de software, a través del análisis del paradigma de desarrollo conocido como MDD (Model Driven Development) integrado con la aplicación de métodos formales y soportados por herramientas automáticas.
El problema es abordado a través de los siguientes sub-objetivos específicos:
Análisis conceptual de las técnicas de modelado de software.
Estudio, elaboración y formalización de lenguajes para crear modelos de software.
Catalogación y formalización de mecanismos de transformación entre modelos.
Definición y formalización de lenguajes para describir transformaciones.
Implementación de herramientas de software automáticas.
  Contexto
  El paradigma MDD tiene dos ejes principales:
- por un lado hace énfasis en la separación entre la especificación de la funcionalidad esencial del sistema y la implementación de dicha funcionalidad usando plataformas tecnológicas específicas. Para ello, el MDD identifica dos tipos principales de modelos: modelos con alto nivel de abstracción e independientes de cualquier tecnología de implementación, llamados PIM (Platform Independent Model) y modelos que especifican el sistema en términos de construcciones de implementación disponibles en alguna tecnología específica, conocidos como PSM (Platform Specific Model);
- por otro lado, los modelos son considerados los conductores primarios en todos los aspectos del desarrollo de software. Un PIM es transformado en uno o mas PSMs, es decir que para cada plataforma tecnológica específica se genera un PSM especifico. La transformación entre modelos constituye el motor del MDD y de esta manera los modelos pasan de ser entidades meramente contemplativas a ser entidades productivas.
La iniciativa MDD cubre un amplio espectro de áreas de investigación: lenguajes para la descripción de modelos, definición de lenguajes de transformación entre modelos, construcción de herramientas de soporte a las distintas tareas involucradas, aplicación de los conceptos en métodos de desarrollo y en dominios específicos, etc.
Actualmente, algunos de estos aspectos están bien fundamentados y se están empezando a aplicar con éxito, otros sin embargo están todavía en proceso de definición. En este contexto son necesarios esfuerzos que conviertan MDD y sus conceptos y técnicas relacionados en una aproximación coherente, basada en estándares abiertos y soportada por técnicas y herramientas maduras.
  Relevancia del Tema
  La aplicación de MDD contribuye a mejorar el proceso de desarrollo de software en los siguientes aspectos:
Productividad: el desarrollo de software basado en modelos mejora la productividad global del proyecto. Se observa que las primeras fases del proceso son menos productivas respecto a otras metodologías (por ejemplo XP), sin embargo esto es compensado en las etapas posteriores, especialmente en la etapa de mantenimiento, ya que introducir modificaciones sobre los modelos es mucho mas rápido y fácil que hacerlo sobre el código.
Portabilidad: en la industria del software surgen nuevas tecnologías cada año. En general resulta necesario y/o conveniente adaptar el software existente a dichos cambios tecnológicos. MDD facilita la portabilidad ya que los modelos son independientes de cualquier tecnología, es decir que el mismo PIM puede ser transformado automáticamente en múltiples PSMs para diferentes plataformas.
Reuso: los modelos abstractos de un sistema de software ofrecen una posibilidad más amplia de reuso. A partir de un modelo es posible derivar toda una familia de sistemas concretos para usos específicos, y utilizando diferentes tecnologías especificas.
Interoperabilidad: la mayoría de los sistemas de software se comunican con otros sistemas de software. Típicamente estos sistemas están implementados con diferentes tecnologías. MDD favorece la interoperabilidad entre sistemas heterogéneos, ya que permite la especificación de las conexiones entre las distintas tecnologías.
  Estado del Arte 
  A continuación se describen los principales avances en cada faceta del MDD:
Modelos y Lenguajes de modelado: el Object Management Group (OMG) ha definido lenguajes de modelado adecuados para especificar PIMs y PSMs. El más conocido y usado es el Unified Modeling Language (UML), acompañado por el Object Constraint Language (OCL) que es un lenguaje de consulta para UML. UML es un lenguaje muy expresivo que permite la creación de distintas modelos del sistema desde diferentes puntos de vista y diferentes niveles de abstracción. UML provee adicionalmente mecanismos de extensión llamados profiles que nos permiten definir lenguajes derivados de UML. Lenguajes definidos mediante profiles son adecuados para crear modelos en dominios específicos. Varios profiles han sido estandarizados por el OMG, por ejemplo profiles para Enterprise Java Beans (EJB), NET, COM y CORBA entre otros. Profiles generalmente se utilizan para defiir PSMs. Por otra parte, para que los modelos puedan ser manipulados automáticamente es necesario que los lenguajes de modelado estén formalmente definidos; por ello, el OMG tiene un lenguaje especial llamado Meta Object Facility (MOF) que se utiliza para definir formalmente a todos los otros lenguajes de modelado.
Transformación de modelos: el OMG esta actualmente trabajando en la definición de un lenguaje estándar para describir transformaciones. Este estándar es conocido como QVT, que quiere decir Query, View and Transformations y se encuentra en proceso de estandarización.
Herramientas automáticas: actualmente existen herramientas que combinan algunas de las funciones requeridas. Estas funciones pueden ser específicas para un lenguaje o pueden ser genéricas. Una herramienta especifica puede por ejemplo proveer un editor de modelos UML y generar código C# a partir de los diagramas UML. Un editor genérico puede permitirle al usuario elegir un lenguaje de modelado de un catálogo y hasta permitirle crear su propio lenguaje de modelado. Una lista bastante completa de herramientas puede encontrarse en la página de OMG sobre MDA.
Aplicación de métodos formales al MDD: acompañando a las actividades de estandarización de lenguajes y procesos coordinadas por el OMG, las cuales tienen un enfoque básicamente “pragmático”, existe una bien organizada comunidad científica, de la cual los integrantes de este proyecto forman parte, abocada al análisis de los elementos del MDD desde la perspectiva científica.
