Resumen del proyecto
Desarrollo de componentes para la moderación digital de reuniones que serán integrados al proyecto Digimod del Instituto Fraunhofer IPSI.
Alcance del proyecto
DIGIMOD es un proyecto del laboratorio de investigación y desarrollo Fraunhofer IPSI (Alemania) cuyo objetivo es el desarrollo de herramientas para llevar a cabo moderaciones digitales de reuniones. Esto es, un moderador utiliza un conjunto de componentes diseñadas para emular actividades que generalmente se realizan de forma manual. Cada participante interactúa mediante una PC con una componente específica seleccionada por el moderador para dicha sesión de moderación (por ejemplo, un componente para recolección de ideas). En todo momento el moderador, mediante otra PC y utilizando una componente de administración, mantiene el control de la sesión.

La participación del LIFIA dentro del proyecto consistió en llevar a cabo el diseño y desarrollo de dos componentes de moderación digital, una componente destinada a generar ideas utilizando la técnica de Brainstorming y la otra para rankear las ideas generadas previamente con la componente anterior. El desarrollo se realizo utilizando un framework construido por el IPSI para tal fin. Dicho framework brinda soporte de comunicación entre componentes, de control de concurrencia, de persistencia, y de recuperación ante fallos entre otras cosas. Las componentes forman parte de un conjunto mucho mayor de componentes interrelacionadas dentro de las aplicaciones de moderación.

Cada componente tiene dos modos de interacción (o vistas), por un lado el modo participante en donde los mismos puede interactuar con la componente que en ese momento esta activa. Y otro modo de mediador donde sólo el coordinador o moderador de la sesión de moderación puede interactuar con la componente mientras que los participantes actúan sólo como observadores.
Tecnologías y metodologías utilizadas
El desarrollo se llevó a cabo en Java jsdk1.4.2. El intercambio de información entre los clientes y los servidores se llevó a cabo con XML. Como herramientas de desarrollo se utilizaron Eclipse 3.0, ANT, CVS como repositorio de código, Mantis para el reporte de Issues y Bugs a realizar, y JUnit para realizar los test de unidad.

La escritura del código fuente se llevo a cabo teniendo en cuenta un estándar de codificación confeccionado por el IPSI que establecía un conjunto de reglas y sugerencias. Todo código subido al repositorio estuvo cubierto al 80% por tests de unidad (Junit).
