Vehicular routing and adjustment tool  

Getxo local government is currently promoting an environmental plan for the recollection and management of urban garbage. The main objective of this plan is to foster garbage recycling by means of selective recollection. This plan relies on the possibility of managing garbage position as well as knowing the exact position of each vehicle (equipped with GPS devices). The problem arises when considering the error margin introduced by this kind of devices (about 10 meters at most).

   
Required functionality

In the context of this project we developed the following bookshops

Digital cartography import

By means of this, we unified the digital cartography provided as DGN (Microstation) and Shapefile (ESRI) files format into a database.
The cartography includes information about blocks, avenues, buildings and garbage recollection points.
Positions adjustment

If we consider the positions reported by the GPS device and the streets and avenues present in the digital cartography, it is clear that there is no exact match (due to the error introduced by the GPS); for this reason, we created a tool that adjusts the positions reported by the GPS to the digital cartography.
Vehicle paths generation
Once adjusted, the vehicles´ positions can be used to generate different digital map files, such as DGN, CGM (Computer Graphics Metafile), SVG (Scalable Vector Graphcis) or JPEG (Joint Photographic Experts Group).
Technology we used
We used Microsoft Visual Studio .Net to develop the dll libraries along with Intergraph´s GeoMedia WebMap 5.1. We have also used MSSQL database server to store the position being adjusted.
